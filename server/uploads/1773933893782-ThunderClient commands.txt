## 🟢 STEP 1: REGISTER USER-1

### 🔹 Request

```
POST http://localhost:5000/api/auth/register
```

### 🔹 Body (JSON)

```json
{
  "name": "User One",
  "email": "user1@gmail.com",
  "password": "123456",
  "skillsOffered": ["Java"],
  "skillsWanted": ["Web Development"]
}
```

### ✅ What this does

* Creates **User-1**
* Stores profile info
* ❌ No matching yet

# 🔥 STEP 5: CREATE SKILL RECORDS (THIS ENABLES MATCHING)

Now comes the **MOST IMPORTANT PART**.

---

## 🔵 STEP 5A: USER-1 ADDS WHAT HE WANTS TO LEARN

### 🔹 Request

```
POST http://localhost:5000/api/skills
```

### 🔹 Headers

```
Authorization: Bearer TOKEN_USER_1
```

### 🔹 Body

```json
{
  "name": "Web Development",
  "category": "Technology",
  "level": "Beginner",
  "type": "learn"
}
```

### ✅ What this creates

```json
Skill {
  user: User-1,
  name: Web Development,
  type: learn
}
```

---

## 🔵 STEP 5B: USER-2 ADDS WHAT HE OFFERS

### 🔹 Request

### 🔹 Headers

### 🔹 Body


### ✅ What this creates
